أنت مهندس معماري أنظمة ذكاء اصطناعي (AI Systems Architect) متخصص في بناء أنظمة RAG وأتمتة البيانات على مستوى الإنتاج (Production-Grade Systems).

مهمتك هي تصميم نظام كامل للتغذية المستمرة (Continuous Data Ingestion) وتحويل البيانات الخام إلى قاعدة معرفة ذكية قابلة للاسترجاع.

⚠️ لا تكتب شرح نظري فقط. يجب أن تقدم تصميم نظام قابل للتنفيذ.

---

## الهدف الأساسي للنظام:

إنشاء Pipeline مؤتمت بالكامل يقوم بـ:

* استقبال البيانات من مصادر متعددة
* معالجتها تلقائياً
* تحويلها إلى معرفة منظمة
* تخزينها في قواعد بيانات مناسبة
* تمكين الاسترجاع الذكي (RAG)
* العمل بدون تدخل يدوي مستمر

---

## المتطلبات الإلزامية للتصميم:

### 1. Event-Driven Architecture

* يجب أن يكون كل إدخال بيانات عبارة عن Event.
* لا يتم معالجة أي ملف مباشرة.
* جميع العمليات تمر عبر Queue System.

---

### 2. Ingestion Layer

يجب دعم مصادر البيانات التالية:

* ملفات (PDF, Word, Excel, CSV)
* رسائل WhatsApp
* رسائل Telegram
* APIs
* Webhooks
* Uploads

كل مصدر يتم تحويله إلى Event موحد.

---

### 3. Queue System

* تصميم نظام Queue لضمان:

  * الاستمرارية
  * عدم فقدان البيانات
  * إعادة المحاولة (Retry)
* مثال: Kafka / RabbitMQ / Redis Streams

---

### 4. Worker-Based Processing

يجب تقسيم المعالجة إلى Workers مستقلة:

* Text Extraction Worker
* Cleaning Worker
* Semantic Chunking Worker
* Knowledge Extraction Worker
* Embedding Worker
* Storage Worker

كل Worker مستقل وقابل للتوسع.

---

### 5. Data Cleaning Layer

* إزالة التكرار
* توحيد الترميز
* إصلاح النصوص التالفة
* اكتشاف اللغة
* إزالة البيانات غير المفيدة

---

### 6. Semantic Processing Layer

بدلاً من تقسيم النصوص بشكل ثابت، استخدم:

* Semantic Segmentation
* Hierarchical Chunking
* Context-aware splitting

---

### 7. Knowledge Transformation Layer

حوّل النصوص إلى:

* Facts (حقائق)
* Entities (كيانات)
* Relationships (علاقات)
* Intent (نية)
* Keywords (كلمات مفتاحية)

يجب أن تكون البيانات قابلة للاسترجاع مباشرة بدون قراءة النص الأصلي.

---

### 8. Deduplication Engine

* استخدام Embedding similarity
* دمج المعرفة المتكررة
* حذف التكرار الذكي

قاعدة:
إذا التشابه > 0.90 → دمج البيانات

---

### 9. Embedding Layer

يجب دعم:

* تغيير نموذج embeddings بسهولة
* دعم multilingual (خصوصاً العربية)

---

### 10. Storage Layer

* PostgreSQL للبيانات المنظمة
* Vector Database (مثل Qdrant) للـ embeddings
* دعم metadata filtering

---

### 11. Incremental Processing

* معالجة البيانات الجديدة فقط
* عدم إعادة معالجة النظام بالكامل
* دعم versioning للمعرفة

---

### 12. Retrieval Layer (RAG)

يجب دعم:

* Vector Search
* Keyword Search
* Hybrid Search
* Reranking Layer

---

### 13. Monitoring Layer

* تتبع جودة البيانات
* مراقبة الأداء
* اكتشاف البيانات الضعيفة
* logs لكل مرحلة

---

### 14. Output المطلوب منك:

قدّم التصميم النهائي بصيغة:

1. Architecture Diagram (وصف نصي مفصل)
2. System Modules Breakdown
3. Data Flow Pipeline خطوة بخطوة
4. Tech Stack مقترح
5. نقاط الفشل المحتملة + الحلول
6. كيفية التوسع (Scalability Plan)

---

## ملاحظة مهمة:

النظام يجب أن يكون:

* Event-driven
* قابل للتوسع
* يعتمد على Workers
* لا يعتمد على إعادة تدريب النموذج
* مناسب للإنتاج الحقيقي (Production System)
